# Работа с файлами

- Создание директории:  
`mkdir`

- Создание файла:  
`touch filename.ext`

- Удаление файла:  
`git rm filename.ext`

   - Удаление файла только из коммита (оставить в каталоге):  
   +`cached`  

- Сменить директорию:  
`cd`

   - Cменить на указанную:
   +`adress/dirname` 

   - Подняться на уровень выше:  
   +`..`

- Вывод списка всех файлов директории:  
`ls`

   - Вывод в виде списка:  
   +`-l` 

   - Вывод в т.ч. скрытых файлов:  
   +`-a`

- Вывод подробностей о файле:  
`git show filename.ext`

- Переименование или перемещение индексированных файлов.  
Базовая команда:  
`git mv`

   - Переименование файла:  
   +`oldname.ext newname.ext`

   - Перемещение файла. Можно сразу же и переименовать:  +`filename.ext /new-dir/filename.ext`


# Работа с репозиторием

- Создание локального репозитория:  
`git init`

- Набор команд по индексации файлов.  
Базовая команда:   
`git add`  

   - Индексация всех отслеживаемых файлов:  
   +` .`

   - Индексация конкретного файла:  
   +` filename.ext`


- Отображение состояния рабочего каталога и раздела проиндексированных файлов:  
`git status`  
С ее помощью можно проверить индексацию изменений и увидеть файлы, которые не отслеживаются Git.


- Открытие репозитория в VS Code:  
`code .`

# Работа с ветками

- Набор команд по управлению ветками.
Базовая команда:  
`git branch`  

   - Вывод всех веток в репозитории (в т.ч. на удаленных серверах):  
   +`-a`  

   - Создание ветки с указанным именем:  
   +`branchname`  

   - Создание новой ветки и переход на нее:  
   +`-b branchname`  

   - Создание новой ветки от коммита:  
   +`-b commithash`  

   - Удаление указанной ветки:  
   +`-d branchname`  

# Объединение веток 

- Набор команд __merge__.  
Объединяет ветки с сохранением истории коммитов интегрируемой ветки и создает результирующий коммит.  
Базовая команда:  
`git merge`  

   - Объединение указанной ветки с текущей:  
   +`branchname`

- Набор команд __rebase__.    
Сжимает все коммиты из интегрируемой ветки один «патч» и интегрирует ее в целевую ветку.  
Базовая команда:  
`git rebase` 

   - Объединение указанной ветки с текущей:  
   +`branchname`

   - Вызов интерактивного rebase:  
   +`-i`

# Работа с коммитами

- Набор команд для создания коммита.  
Базовая команда:  
`git commit`  

   - Добавить коммиту сообщение:  
   +`m 'message'`

   - Добавить в индекс все файлы из версионного контроля и сделать коммит (git add . + git commit):  
   +`-a`

   - Добавить проиндексированные данные к последнему коммиту не изменяя его название:  
   +`--amend --no-edit`

- Набор команд по анализу истории коммитов.  
Базовая команда (выводит историю):  
`git log`

   - Вывод истории коммитов с подробностями:  
   +`-p`

- Вывод подробностей о коммите:  
`git show commit-hash`  

- Дублирование коммита из одной ветки в другую:  
`git chery-pick commithash`  

- Набор команд по работу с тегами коммита.  
Базовая команда (Вывод списка присвоенных коммиту тегов):  
`git tag`

   - Добавление аннотированного тега:  
   +`-a tagname tagmessage`

   - Добавление легковесного тега:  
   +`tagname`

# Управление HEAD

- Набор команд __checkout__.    
Базовая команда:  
`git checkout` 

   - Переключение на указанную ветку:
   +`branchname`

   - Переключение на указанную ветку и ее создание в случае отсутствия:  
   +`-b branchname`

   - Переключение на указанный коммит:  
   +`commithash`

   - Откат неиндексированных изменений в файле на состояние последнего коммита (аналог git restore):  
   +`filename.ext`

- Набор команд __reset__.  
Базовая команда:  
`git reset`  

   - Hard-режим:  
   +`--hard`

      - Перевод head на указанный коммит и удаление всех внесенных после него изменений:  
      +`commithash`

      - Перевод head в указанную ветку c удалением файлов из рабочей директории активной ветки:  
      +`branchname`

   - Mixed-режим:  
   +`--mixed`  

      - Перевод head на указанный коммит и перевод всех внесенных после него изменений в статус untracked:  
      +`commithash`

      - Перевод head в указанную ветку c добавлением в not-staged-файлы (не staging area) данных из staging area активной ветки:  
      +`branchname`

   - Soft-режим:  
   +`soft`

      - Перевод head на указанный коммит с сохранением пометки staged во всех внесенных после него изменениях:  
      +`commithash`

      - Перевод head в указанную ветку c добавлением в ее staging area данных из staging area активной ветки:  
      +`branchname`

# Работа со staging area  

- Откат неиндексированных изменений в состояние последнего коммита:  
`git restore filename.ext`

   - Откат индексированных изменений в неиндексированное состояние:  
   +` --staged`  
   
- Вывод неиндексированных изменений, внесенных в отслеживаемые файлы после последнего коммита:  
`git diff`

   - Вывод изменений в индексе, внесенных после последнего коммита:  
   +`--staged`

   - Отслеживание конкретного файла:  
   +`filename.ext`

# Работа с сервером

- Набор команд по отправке данных в удаленный репозиторий.  
Базовая команда:  
`git push` 

   -  Отправка в конкретный репозиторий и ветку:  
   +`origin branchname`  

   - Отправка конкретного тега (по умолчанию не отправляются):  
   +`origin tagname`  

   - Отправка всех тегов (по умолчанию не отправляются):  
   +`--tags`

- Набор команд по приему данных из удаленного репозитория.  
Базовая команда:  
+`git pull` 

   - Прием из конкретного репозитория и ветки:  
   +`origin branchname`  

- Клонирование репозитория из удаленного сервера:  
+`git clone` 

   - Через ssh-ключ:  
   +`ssh-key`

- Набор команд по управлению удаленным сервером.  
Базовая команда:  
`git remote`

   - Добавление удаленного репозитория к локальному (связывание локального репозитория с удаленным):  
   +`add origin ssh-key`  

   - Вывод связанных с проектом удаленных репозиториев:  
   +`-v`

- Обновление ветки слежения и скачивание недостающих коммитов:  
`git fetch origin`  
При этом head остается в том же коммите. Нужно выполнить команду merge для объединения своей локальной ветки с удаленной (git merge)
