# INHERITANCE ARCHITECTURE

## About
- В JS наследование реализуется через цепочку прототипов: у каждого объекта есть один (строго!) объект-прототип.
- В каждом объекте есть скрытое свойство [[Prorotype]], которое ссылается на свой объект-прототип, либо имеет значение null (если объекта-прототипа нет).
- Манипулировать свойством [[Prototype]] можно через:
  - __ Proto __ - геттер/сеттер для [[Prototype]]
  - Object.getPrototypeOf/setPrototypeOf
- В объекте-прототипе объект может: 
  - читать свойства (унаследованные свойства).
  - использовать методы.
- В объекте-прототипе объект НЕ может:
  - производить запись/удаление (исключение - свойства-аксессоры).
- Контекст вызова метода из прототипа - всегда текущий объект.

## Манипулирование прототипом через __proto__
- Назначить прототип:

  ```
  obj.__proto__ = objPrototype;
  ```
- Ограничения:
  - Ссылки `__proto__` не должны идти по кругу.
  - Значением `__proto__` должны быть только объект или null.

## Methods
- Является ли свойство - своим (а не прототипа)?

  ```
  obj.hasOwnProperty(key)
  ```
